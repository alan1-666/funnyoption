## 版本目标

v1.2 聚焦三条主线：

1. **CI 基础设施** — 建立 PR 级别的自动化测试门禁
2. **前端交易体验** — Market Order、Portfolio 真实数据、WebSocket 实时价格
3. **后端稳定性** — 全服务统一 healthcheck + graceful shutdown

## 功能交付

### A. CI Pipeline

| 项目 | 内容 |
|------|------|
| 文件 | `.github/workflows/ci.yml` |
| 触发 | `pull_request` + `push: main` |
| Job 1 | `go-test` — `go vet ./...` + `go test ./...` |
| Job 2 | `forge-test` — `forge test`（Solidity 合约） |
| Job 3 | `web-build` — `npm ci && npm run build`（TypeScript 编译验证） |

所有后续改动都在自动测试保护下进行。

### B1. Market Order

**后端 (`freeze.go` + `order_handler.go`)**

- 在 API 层白名单 `LIMIT` / `MARKET` 两种 order type
- `MARKET` 订单自动强制为 `time_in_force=IOC`，`price=100`
- Freeze 逻辑：BUY MARKET 冻结 `100 × quantity`（最大可能成本）
- 撮合引擎无需改动：IOC price=100 等价于 market order 语义
- 新增 `TestCalculateFreezeBuyMarketOrder` / `TestCalculateFreezeSellMarketOrder` 单元测试

**前端 (`order-ticket.tsx`)**

- 新增 Limit / Market 切换 tab
- Market 模式隐藏价格输入、显示"最大冻结"
- 签名和提交逻辑适配 order type

### B2. Portfolio 真实数据

- `portfolio-shell.tsx` 接入 `session.userId` 触发 fetch
- 复用已有 API（balances、positions、orders、payouts）
- 与 ShellTopBar 余额修复共用相同的 session 状态管理模式

### B3. WebSocket 实时价格推送 (bonus, 超出原计划)

**`useTicker` hook (`web/hooks/use-ticker.ts`)**

- 单例共享模式：同一 market 的多个组件共用同一对 YES/NO WebSocket
- 引用计数管理：最后一个消费者卸载时自动关闭连接
- 返回 `{ yes, no, connected }`，含 `lastPrice`、`bestBid`、`bestAsk`

**OrderTicket 实时价格**

- YES/NO 按钮和参考价格区域实时更新
- 优先级链：`lastPrice > midpoint(bestBid, bestAsk) > bestBid/bestAsk > SSR 初始值`

**市场详情 Hero 实时价格 (`LiveOutcomePrice`)**

- 客户端组件嵌入服务端渲染的 Hero 区域
- 与 OrderTicket 共享同一 WS 连接

### C1. 统一 Healthcheck + Graceful Shutdown

**shared/health 包 (`backend/internal/shared/health/`)**

- `ListenAndServe(ctx, logger, addr, service, env)` — 一行接入
- 自动暴露 `GET /healthz`，返回 `{"status":"ok","service":"...","env":"..."}`
- ctx 取消时 graceful shutdown（3s 超时）

**healthcheck 探针 (`backend/cmd/healthcheck/`)**

- 静态编译的 Go HTTP 探针
- 解决 distroless 容器没有 wget/curl 的问题
- 用法：`/app/healthcheck http://localhost:PORT/healthz`

**全服务覆盖**

| 服务 | /healthz 方式 | Health 端口 |
|------|-------------|------------|
| api | 已有 (Gin) | 8080 |
| ws | 已有 (Gin) | 8081 |
| oracle | 已有 (stats.go) | 9191 |
| matching | 新增到 HA HTTP mux | 9190 |
| account | shared/health | 8091 |
| settlement | shared/health | 8093 |
| chain | shared/health | 8094 |
| ledger | shared/health | 8095 |
| notification | shared/health | 8096 |
| market-maker | shared/health | 8097 |

**config 默认 HTTP 端口**

- `config.go` 的 `defaultHTTPAddr()` 为所有 10 个服务提供默认 HTTP 地址
- 无需手动配置环境变量即可启用 health server

**Graceful Shutdown**

- 所有 10 个 cmd/*/main.go 已确认使用 `signal.NotifyContext`
- chain 是本版本新增（此前缺失）
- `grpcx.Run` 已有 `GracefulStop` + 5s 超时降级机制

**Docker 改动**

- 10 个 Go Dockerfile 更新：同时构建 healthcheck 二进制并 COPY 到容器
- `docker-compose.staging.yml`：
  - 所有 Go 服务加 `healthcheck` 配置
  - 所有 `depends_on` 从 `service_started` 升级为 `service_healthy`
  - web/admin 等待 api + ws healthy 后再启动
